TDE 3 - Rede Para Empresa De Varejo Com Multiloja

Projeto - Arquitetura de Rede para Empresa de Varejo com Multiloja

Para uma empresa de varejo com múltiplas lojas, é crucial identificar os requisitos do sistema para garantir que a rede suporte as operações diárias. Os dados manipulados incluem:

  • Transações de vendas dos sistemas de ponto de venda (PDV)
  • Níveis de estoque e atualizações em tempo real
  • Informações de clientes, como cadastros e programas de fidelidade
  • Registros de funcionários, incluindo dados de RH
  • Dados financeiros e contábeis para relatórios gerenciais.

IP (Internet Protocol)

O IP é o protocolo fundamental da internet que permite o endereçamento e roteamento de pacotes de dados entre diferentes redes. Atua na camada da rede ( camada 3 ) do modelo OSI, atribuindo endereços únicos e definido como os pacotes são encaminhados até seu detino.

Características Principais:

Sem conexão

Baseado em melhor esforço

Versões IPv4 e IPv6

Cada pacote é tratado
independentemente

Não garante entrega ou ordem
dos pacotes

Opera com IPv4 (32 bits)
ou IPv6 (128 bits)

TCP (Transmission Control Protocol)

O TCP é um protocolo da camada de transporte ( camada 4 ) que estabelece conexões entre aplicativos. garante que todos os pacotes sejam entregues sem erros e na sequência correta, sendo ideal para dados que não podem ser pedidos

Características Principais:

Orientado à conexão

Estabelece, mantém e encerra conexões

Recuperação de erros

Retransmite pacotes perdidos

Controle de fluxo

Evita sobrecarga do receptor

Controle de congestionamento

Ajusta a taca de transferencia conforme condições da rede

HTTP (Hypertext Transfer Protocol)

HTTP é um protocolo da camada de aplicação usado para transmissão de hipermídia, sendo base para comunicação de dados na web muitas interfaces de sistemas internos. Opera no modelo requisição-resposta entre cliente e servidores.

Requisição Cliente

Cliente envia solicitação ao servidor

Renderização

Cliente exibe o conteúdo recebido

Processamento

Servidor processa a requisição

Resposta Servidor

Servidor envia dados solicitados

Características Principais:

  • Stateless
  • Métodos diversos
  • Extensível
  • Versões: HTTP/1.1, HTTP/2, HTTP/3

DNS (Domain Name System)

DNS é um sistema hierárquico e distribuido que traduz nomes de domínio amigáveis em endereços IP. Funciona como a "lista telefonica" da internet, permitindo que o usuário acessem recursos usando nomes em vez de números.

Características Principais:

  • Estrutura hierárquica
  • Mecanismo de cache
  • Sistema distribuído
  • Diversos tipos de registros

SMTP e IMAP

SMTP (Simple Mail Transfer Protocol)

SMTP é o protocolo padrão para transmissão de e-mails entre servidores de correio eletrõnico. Define como as mensagens são enviadas, roteadas e entregues em redes TCP/IP.

  • Confiável no envio de mensagens
  • Opera na porta 25 (padrão)
  • Baseado em filas de mensagens
  • Inclui mecanismo para notificação de entrega/falha
  • Extensões modernas para autenticação e segurança

IMAP (Internet Message Access Protocol)

IMAP permite que clientes de e-mail acessem mensagens armazenadas em um servidor remoto. Diferente do POP3, mantém as mensagens no servidor, permitindo acesso sincronizado de múltiplos dispositivos.

  • Acesso às mensagens permanentemente
  • Gerenciamento de pastas no servidor
  • Suporte a múltiplos dispositivos simultâneos
  • Download parcial de mensagens (cabeçalho, corpo)
  • Estados de leitura sincronizados entre dispositivos
  • Opera na porta 143 (com TLS)

Aplicações P2P (Peer-to-Peer)

Características:

Descentralização

Não existe um ponto único de
controle ou falha

Conexão direta

Comunicação direta entre pares
sem intermiediários

Escalabilidade

Quanto mais participantes, mais
recursos disponíveis

Resiliência

Continua funcionando mesmo sem
alguns nós ficarem indisponíveis

Resposta Servidor

Recursos e processamento divídidos
entre participantes

Redes P2P são sistemas distribuidos onde os participantes (nós ou "peers") compartilham resursos diretamente entre si, sem depender de um servidor central. Cada participante pode funcionar simultaneamente como cliente e servidor, contribuindo com resursos e consumindo serviços de rede

Tipos de Arquiteturas P2P:

Pura:

Completamente descentralizada, sem servidores (ex: BitTorrent)

Híbrida:

Combina elementos centralizados e P2P (ex: Skype)

Não estruturada:

Conexôes mais orgânicas entre participantes

Estruturada:

Organizada com tabelas de hash distribuídas (DHT)
